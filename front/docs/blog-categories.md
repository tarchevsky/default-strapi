# Добавление категорий блога на фронте

Статьи отображаются по адресу `/blog/[category-slug]/[article-slug]`. Категория в Strapi (например «Методические материалы») маппится в URL-слаг (например `metodika`).

## Где править

Файл: **`src/types/page.types.ts`**.

## Шаги

1. **Добавить значение в тип `PageCategory`**  
   Новая строка в union-типе, в точности как в enum категорий в Strapi.

2. **Добавить в массив `PAGE_CATEGORIES`**  
   Чтобы категория участвовала в фильтрах и селекте в сайдбаре блога.

3. **Добавить запись в `CATEGORY_SLUG_MAP`**  
   Ключ — точное название категории (как в Strapi), значение — латинский слаг для URL (без пробелов, в нижнем регистре).

Обратный маппинг (слаг → категория) и хелперы `getCategorySlug` / `getCategoryBySlug` строятся из `CATEGORY_SLUG_MAP` автоматически, их менять не нужно.

## Пример

Добавляем категорию «Кейсы» со слагом `cases`:

```ts
// 1. В тип PageCategory
export type PageCategory =
  | 'Методические материалы'
  | 'Глоссарий'
  | 'Статья'
  | 'Поддержка'
  | 'Мысли'
  | 'Кейсы'

// 2. В PAGE_CATEGORIES
export const PAGE_CATEGORIES: PageCategory[] = [
  'Методические материалы',
  'Глоссарий',
  'Статья',
  'Поддержка',
  'Мысли',
  'Кейсы',
]

// 3. В CATEGORY_SLUG_MAP
export const CATEGORY_SLUG_MAP: Record<PageCategory, string> = {
  'Методические материалы': 'metodika',
  Глоссарий: 'glossary',
  Статья: 'article',
  Поддержка: 'support',
  Мысли: 'thoughts',
  Кейсы: 'cases',
}
```

После этого в Strapi нужно добавить то же значение в enum поля **Category** у контент-типа Page (для типа страницы «статья»).
